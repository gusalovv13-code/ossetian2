# Изменения

## 1.1.0 — первый этап защиты

- Добавлена серверная проверка подписи `Telegram.WebApp.initData`.
- Сервер больше не принимает `ownerId`, `userId`, имя и username пользователя из клиентских запросов.
- Удаление объявления выполняется только владельцем из проверенной Telegram-сессии.
- Избранное и список своих объявлений привязаны к проверенному Telegram ID.
- Аватар Telegram проксируется как изображение; URL с `BOT_TOKEN` больше не попадает в браузер.
- Клиент получает профиль через защищённый маршрут `/api/me` и не доверяет `initDataUnsafe`.
- Публичные файлы перенесены в папку `public`; серверные исходники больше не раздаются статически.
- Идентификаторы новых объявлений создаются через `crypto.randomUUID()`.
- Удалены дублирующая функция телефона и неиспользуемый клиентский генератор ID.
- `package-lock.json` синхронизирован, Docker переведён на `npm ci` и запуск не от root.
- Добавлены автоматические тесты Telegram-аутентификации.
