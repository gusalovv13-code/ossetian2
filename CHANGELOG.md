# Changelog

## 1.11.6

- В админке рекламы URL изображения заменён на загрузку фото с телефона или компьютера.
- Добавлены предпросмотр, удаление и автоматическое сжатие рекламного изображения.
- Base64-фото рекламы больше не передаётся внутри публичного JSON: браузер получает отдельный кешируемый URL изображения.
- Сервер проверяет реальный формат JPEG, PNG и WEBP, ограничивает размер и возвращает понятные JSON-ошибки для слишком больших или повреждённых данных.
- Доступ к localStorage и sessionStorage защищён для WebView, где браузерное хранилище может быть недоступно.
- Для сломанных рекламных изображений добавлен безопасный резервный рисунок.

## v1.11.4
- Исправлено расписание рекламы: значения `datetime-local` теперь переводятся в ISO с учётом часового пояса пользователя.
- Первая реклама в ленте показывается даже при количестве товаров меньше заданного интервала вставки.
- Публичный API рекламы больше не кешируется браузером или прокси после изменения кампании.
- В админке добавлена понятная причина, почему кампания показывается или не показывается.
- Добавлены регрессионные проверки исправлений рекламы.

## v1.11.2
- Base64-фотографии удалены из JSON каталога, избранного, профиля продавца и публичной карточки товара.
- Добавлены отдельные кешируемые маршруты миниатюр и полноразмерных изображений.
- Главный экран больше не запускает загрузку каталога в фоне, поэтому кнопки остаются отзывчивыми после запуска приложения.
- Убран отдельный `COUNT(*)` из пагинации каталога: наличие следующей страницы определяется через `LIMIT + 1`.
- Настроены тайм-ауты и пул подключений PostgreSQL.
- Добавлены стили `touch-action: manipulation` и изоляция отрисовки карточек.
- Добавлено понятное состояние повторной загрузки при сетевой ошибке.
- Добавлены регрессионные тесты для компактных JSON-ответов и ленивого запуска каталога.

## v1.11.1
- Ускорен каталог: первая страница уменьшена до 12 карточек и добавлена кнопка «Показать ещё».
- Публичные списки больше не передают все фотографии и полные характеристики каждого объявления.
- Добавлены отдельные миниатюры для новых и отредактированных объявлений.
- Клиент использует WebP для новых фотографий и миниатюр, когда формат поддерживается браузером.
- Убрана повторная передача полного объявления при увеличении счётчика просмотров.
- Добавлено кеширование каталога, избранного, собственных объявлений и открытых карточек.
- Старые запросы поиска отменяются, поэтому быстрый ввод не создаёт очередь сетевых обращений.
- «Мои объявления» и полный список избранного переведены на ленивую загрузку.
- Переключение страницы выполняется до запуска запросов, плавная прокрутка при переходе убрана.
- Добавлены скелетоны загрузки, приоритет первых изображений и оптимизация отрисовки карточек.
- Добавлено gzip-сжатие ответов и кеширование статических файлов.
- Добавлен индекс PostgreSQL для публичной ленты.
- Добавлены 8 регрессионных тестов производительности.

## v1.11.0
- Добавлена история изменений цены и автоматическая отметка «Цена снизилась» со старой ценой и процентом снижения.
- Добавлена таблица истории цен и отображение последних снижений в карточке товара.
- Добавлена автоматическая проверка ссылок, доменов, email, телефонов, Telegram username и запрещённых выражений.
- Заблокированные публикации не попадают в каталог, сохраняются владельцу для исправления и поступают в очередь модератора.
- Добавлено управление запрещёнными словами, фразами и доменами из админ-панели.
- Добавлены переключатели проверки ссылок, контактов и email.
- Администратор может одобрить автоматическую блокировку или окончательно отклонить объявление.
- Заблокированное объявление не может быть самостоятельно возвращено владельцем в публичный каталог.
- Добавлены рекламные кампании для верхнего баннера каталога, вставок в ленте и карточки товара.
- Для рекламы поддерживаются расписание, приоритет, частота вставок, лимит показов и переход на внешнюю HTTPS-ссылку либо объявление.
- Добавлен учёт уникальных дневных показов и кликов, CTR и рекламной статистики.
- Добавлены модели монетизации: фиксированная стоимость, CPM и CPC.
- Админка рассчитывает рекламный доход и хранит статус оплаты кампании.
- Рекламные блоки всегда имеют явную маркировку «Реклама».
- Добавлены автоматические миграции PostgreSQL и регрессионные тесты новых функций.
