# Алания Маркет 2026

Telegram Mini App локального маркетплейса на Node.js, Express и PostgreSQL.

## Структура

- `public/` — единственная рабочая версия клиентских HTML, CSS и JavaScript.
- `server.js` — API, инициализация PostgreSQL и проксирование Telegram-аватара.
- `telegram-auth.js` — проверка подписи `Telegram.WebApp.initData`.
- `test/` — тесты авторизации и целостности основных маршрутов проекта.
- `Dockerfile` — production-сборка контейнера.

## Переменные окружения

Скопируйте `.env.example` в `.env` и заполните:

- `BOT_TOKEN` — токен Telegram-бота.
- `DATABASE_URL` — строка подключения PostgreSQL.
- `DATABASE_SSL` — `true` для облачной БД с SSL или `false` для локального PostgreSQL без SSL.
- `PORT` — порт приложения, по умолчанию `3000`.
- `TELEGRAM_AUTH_MAX_AGE_SECONDS` — максимальный возраст `initData`, по умолчанию 24 часа.
- `ADMIN_TELEGRAM_IDS` — Telegram ID администраторов через запятую.
- `SUPPORT_USERNAME` — Telegram username поддержки без символа `@`.

## Запуск

```bash
npm ci
npm start
```

Проверка синтаксиса и тесты:

```bash
npm run check
npm test
```

## API

Публичные маршруты чтения:

- `GET /api/health`
- `GET /api/products` — поддерживает `q`, `category`, `page` и `limit`.
- `GET /api/products/:id/details` — полная карточка, похожие объявления и другие товары продавца.
- `GET /api/config`
- `GET /api/users/:id`
- `GET /api/users/:id/products`
- `POST /api/products/:id/view`

Маршруты, требующие Telegram-авторизацию:

- `GET /api/me`
- `GET /api/avatar`
- `GET /api/my-products`
- `POST /api/products`
- `PATCH /api/products/:id` — редактирование собственного объявления.
- `PATCH /api/products/:id/status` — статусы `active`, `sold` и `draft`.
- `DELETE /api/products/:id` — мягкое удаление без физического удаления строки.
- `GET /api/favorites`
- `POST /api/favorites`
- `POST /api/products/:id/reports` — отправка жалобы на объявление.

Админ-панель доступна только пользователям из `ADMIN_TELEGRAM_IDS`. Она автоматически
загружает свежие данные, позволяет скрывать объявления, блокировать и разблокировать
пользователей, рассматривать жалобы, искать записи, смотреть журнал действий и аналитику
за 14 дней.

## Возможности карточки объявления

- полноэкранное увеличение фотографий;
- свайп, стрелки, миниатюры и счётчик текущего изображения;
- даты публикации и обновления;
- количество просмотров и добавлений в избранное;
- состояние, торг, доставка, район и произвольные характеристики;
- прямые ссылки, системная кнопка «Поделиться» и копирование ссылки;
- жалоба на объявление;
- другие объявления продавца и похожие товары.

## Проверка качества объявления

При создании и редактировании приложение рассчитывает качество объявления по заполненности
названия, описания, фотографий, категории, цены, состояния, района и характеристик. Пользователь
видит оценку и конкретные рекомендации до публикации. Сервер пересчитывает показатель независимо
от браузера.

## Жалобы и модерация

Пользователь не может пожаловаться на собственное объявление или создать несколько одновременно
ожидающих жалоб на один товар. В админке есть отдельная очередь, где модератор может отклонить
жалобу, скрыть объявление, заблокировать продавца либо выполнить оба действия. Решения сохраняются
в журнале администратора.

Клиент передаёт подписанную строку Telegram в заголовке:

```text
Authorization: tma <Telegram.WebApp.initData>
```

Сервер проверяет подпись и сам определяет пользователя. ID, имя и username владельца из тела запроса не принимаются.

## Статусы объявлений

- `active` — видно в публичном каталоге.
- `sold` — видно только владельцу во вкладке «Проданные».
- `draft` — видно только владельцу во вкладке «Черновики».
- `deleted` — архивная запись, недоступная в пользовательском интерфейсе.

Публичные маршруты возвращают только активные и не скрытые модератором объявления.
