# Алания Маркет 2026 — v1.11.2

Telegram Mini App локального маркетплейса на Node.js, Express и PostgreSQL.

## Структура

- `public/` — единственная рабочая версия клиентских HTML, CSS и JavaScript.
- `server.js` — API, инициализация PostgreSQL и проксирование Telegram-аватара.
- `telegram-auth.js` — проверка подписи `Telegram.WebApp.initData`.
- `test/` — тесты авторизации и целостности основных маршрутов проекта.
- `Dockerfile` — production-сборка контейнера.

## Переменные окружения

Скопируйте `.env.example` в `.env` и заполните:

- `BOT_TOKEN` — токен Telegram-бота.
- `DATABASE_URL` — строка подключения PostgreSQL.
- `DATABASE_SSL` — `true` для облачной БД с SSL или `false` для локального PostgreSQL без SSL.
- `PORT` — порт приложения, по умолчанию `3000`.
- `TELEGRAM_AUTH_MAX_AGE_SECONDS` — максимальный возраст `initData`, по умолчанию 24 часа.
- `ADMIN_TELEGRAM_IDS` — Telegram ID администраторов через запятую.
- `SUPPORT_USERNAME` — Telegram username поддержки без символа `@`.

## API

Публичные маршруты чтения:

- `GET /api/health`
- `GET /api/products` — поддерживает `q`, `category`, `page` и `limit`.
- `GET /api/products/:id/details` — текстовые данные карточки, история цены, похожие объявления и другие товары продавца.
- `GET /api/products/:id/thumbnail` и `GET /api/products/:id/media/:index` — отдельная кешируемая выдача изображений.
- `GET /api/ads` — активные рекламные кампании по размещению.
- `GET /api/ads/:id/image` — оптимизированная выдача загруженного изображения рекламы.
- `POST /api/ads/:id/impression` и `POST /api/ads/:id/click` — учёт рекламной статистики.
- `GET /api/config`
- `GET /api/users/:id`
- `GET /api/users/:id/products`
- `POST /api/products/:id/view`

Маршруты, требующие Telegram-авторизацию:

- `GET /api/me`
- `GET /api/avatar`
- `GET /api/my-products` — облегчённый список собственных объявлений.
- `GET /api/my-products/:id/details` — полные данные собственного объявления для редактирования.
- `POST /api/products`
- `PATCH /api/products/:id` — редактирование собственного объявления.
- `PATCH /api/products/:id/status` — статусы `active`, `sold` и `draft`.
- `DELETE /api/products/:id` — мягкое удаление без физического удаления строки.
- `GET /api/favorites/ids` — компактный список ID избранного для сердечек каталога.
- `GET /api/favorites` — облегчённый список избранных объявлений.
- `POST /api/favorites`
- `POST /api/products/:id/reports` — отправка жалобы на объявление.

Админ-панель доступна только пользователям из `ADMIN_TELEGRAM_IDS`. Она автоматически
загружает свежие данные, позволяет скрывать объявления, блокировать и разблокировать
пользователей, рассматривать жалобы, искать записи, смотреть журнал действий и аналитику
за 14 дней, управлять автоматической модерацией и рекламными кампаниями.


## Производительность v1.11.2

- каталог загружает по 12 облегчённых карточек вместо 100 объявлений со всеми фотографиями;
- Base64-фотографии полностью убраны из JSON публичных списков и карточки: браузер получает их отдельными кешируемыми запросами;
- каталог не загружается на главном экране в фоне и запускается только после открытия раздела;
- пагинация использует `LIMIT + 1` вместо отдельного тяжёлого `COUNT(*)` для каждой страницы;
- кнопка «Показать ещё» догружает следующую страницу без повторной загрузки уже полученных карточек;
- полные фотографии, описание и характеристики загружаются только при открытии объявления;
- новые объявления сохраняют отдельную миниатюру 360 px, а полноразмерные изображения кодируются в WebP, если браузер это поддерживает;
- повторное открытие разделов использует 30-секундный кеш, а карточки товара — 60-секундный кеш;
- старые поисковые запросы отменяются через `AbortController`;
- «Мои объявления» и полное избранное загружаются только после открытия соответствующей страницы;
- учёт просмотра возвращает только ID и новое количество просмотров, а не весь товар с фотографиями;
- переход между страницами выполняется до запуска сетевых запросов;
- текстовые ответы и статические файлы сжимаются и кешируются.

Старые объявления без сохранённой миниатюры продолжают работать: сервер временно использует их основное изображение. После редактирования такого объявления миниатюра будет создана автоматически.

## Возможности карточки объявления

- полноэкранное увеличение фотографий;
- свайп, стрелки, миниатюры и счётчик текущего изображения;
- даты публикации и обновления;
- количество просмотров и добавлений в избранное;
- состояние, торг, доставка, район и произвольные характеристики;
- прямые ссылки, системная кнопка «Поделиться» и копирование ссылки;
- жалоба на объявление;
- другие объявления продавца и похожие товары;
- отметка «Цена снизилась», старая цена, процент снижения и история изменений.

## Проверка качества объявления

При создании и редактировании приложение рассчитывает качество объявления по заполненности
названия, описания, фотографий, категории, цены, состояния, района и характеристик. Пользователь
видит оценку и конкретные рекомендации до публикации. Сервер пересчитывает показатель независимо
от браузера.

## Жалобы и ручная модерация

Пользователь не может пожаловаться на собственное объявление или создать несколько одновременно
ожидающих жалоб на один товар. В админке есть отдельная очередь, где модератор может отклонить
жалобу, скрыть объявление, заблокировать продавца либо выполнить оба действия. Решения сохраняются
в журнале администратора.

Клиент передаёт подписанную строку Telegram в заголовке:

```text
Authorization: tma <Telegram.WebApp.initData>
```

Сервер проверяет подпись и сам определяет пользователя. ID, имя и username владельца из тела запроса не принимаются.

## Статусы объявлений

- `active` — видно в публичном каталоге.
- `sold` — видно только владельцу во вкладке «Проданные».
- `draft` — видно только владельцу во вкладке «Черновики».
- `deleted` — архивная запись, недоступная в пользовательском интерфейсе.

Публичные маршруты возвращают только активные и не скрытые модератором объявления.


## Автоматическая модерация

При создании и редактировании сервер проверяет название, описание, район и характеристики.
По умолчанию блокируются внешние ссылки, домены, email, номера телефонов и Telegram username,
если они вставлены в текст объявления. Телефон продавца следует указывать в отдельном поле.

Администратор может:

- включать и отключать проверку ссылок, контактов и email;
- добавлять запрещённые слова, фразы и домены;
- временно выключать или удалять правила;
- просматривать очередь автоматических блокировок;
- одобрять исправное объявление или окончательно отклонять его.

Заблокированная публикация сохраняется владельцу как черновик, но не попадает в каталог.
Владелец видит причину и может отредактировать текст.

## Реклама и монетизация

Рекламные кампании создаются в админ-панели и не маскируются под пользовательские объявления.
Каждый блок имеет заметную отметку «Реклама». Поддерживаются размещения:

- верх каталога;
- вставки между товарами;
- карточка объявления.

Для кампании задаются изображение, текст, кнопка, внешняя HTTPS-ссылка или ID объявления,
даты начала и окончания, приоритет, частота вставки и лимит показов. Система считает показы,
клики и CTR.

Поддерживаются модели оплаты:

- `flat` — фиксированная стоимость размещения;
- `cpm` — тариф за тысячу показов;
- `cpc` — тариф за переход.

Админка рассчитывает ожидаемый рекламный доход и хранит отметку, оплачена ли кампания.
Фото рекламной кампании выбирается с телефона или компьютера и автоматически сжимается перед сохранением.
Это учётная система, а не встроенный платёжный шлюз: оплату рекламодатель пока производит отдельно.
