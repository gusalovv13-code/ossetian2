# Осетинский маркет v1.12.2

## Исправлена фактическая причина отсутствия фотографий

На существующем сервере таблица `product_images` могла быть создана более старой версией без столбца `preview_url`. Команда `CREATE TABLE IF NOT EXISTS` этот столбец не добавляла. Поэтому запросы миниатюр завершались ошибкой PostgreSQL, а интерфейс показывал постороннюю резервную фотографию или сломанные миниатюры.

В версии 1.12.2:

- добавлена повторяемая миграция `ALTER TABLE product_images ADD COLUMN IF NOT EXISTS preview_url`;
- также проверяются столбцы `position` и `created_at`;
- индекс фотографии в SQL явно приводится к `int`;
- повреждённая запись из `product_images` больше не блокирует корректное фото из `products.images` или `products.image`;
- декодирование base64-фотографий проходит через единую проверку формата и сигнатуры;
- браузер один раз повторяет запрос фото с обходом кеша;
- удалена вводящая в заблуждение стоковая фотография — при окончательной ошибке показывается нейтральная заглушка.

## Установка

1. Сделать резервную копию базы PostgreSQL.
2. Заменить файлы приложения версией 1.12.2.
3. Полностью пересобрать и перезапустить контейнер/процесс Node.js.
4. Убедиться в логах запуска, что миграции выполнились без ошибок.
5. Полностью закрыть Telegram Mini App и открыть заново.

Старые объявления переопубликовывать не требуется: сервер читает фото и из новой таблицы, и из прежних полей базы.
